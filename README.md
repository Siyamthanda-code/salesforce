# Salesforce Handbook: A Comprehensive Tutorial Reference
![License](https://img.shields.io/badge/license-MIT-blue?style=flat-square)

## 📖 Overview

The **Salesforce Handbook** is a static web application designed to be a complete, offline-accessible reference guide for Salesforce Developers and Administrators. It compiles 24 essential topics ranging from basic CRM concepts to advanced Apex programming and Lightning development.

Whether you are a beginner preparing for certification or an experienced professional needing a quick refresher, this handbook provides structured, searchable content in a clean, modern interface.

**Key Features:**
*   **24 Comprehensive Tabs:** Covering everything from Introduction to Batch Apex.
*   **Static Site:** Built purely with HTML, CSS, and JavaScript. No server-side database required.
*   **Fast & Responsive:** Optimized for both desktop and mobile devices.
*   **Clean UI:** Sidebar navigation for easy topic switching.
*   **Interactive Tables & Lists:** Formatted data for easy consumption.

## 🚀 Getting Started

There are two ways to view this project:

### Option 1: View Live on GitHub Pages
You can view the site directly at:
`[https://[your-username].github.io/[repo-name]/](https://siyamthanda-code.github.io/salesforce/)`

### Option 2: Run Locally
1.  **Clone the repository:**
    ```bash
    git clone https://github.com/siyamthanda-code/salesforce.git
    ```
2.  **Open the file:**
    Navigate to the project folder and open `index.html` in your preferred web browser.
    *   *Note: You do not need a local server (like Apache or Node.js) to run this; the files are static.*

## 🛠️ Built With

*   **HTML5**: Structure and content.
*   **CSS3**: Styling, Grid layouts, and Responsive Design.
*   **Bootstrap 5**: UI Framework for tables, cards, and responsive navigation.
*   **JavaScript**: Tab switching logic and dynamic content handling.

## 🎨 Customization

This project is designed to be easy to edit.
*   **To change content:** Edit the specific `<div class="content-section">` blocks in `index.html`.
*   **To change styles:** Modify the `<style>` section in the `<head>` of `index.html` or link an external CSS file.
