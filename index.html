Salesforce Introduction

As customers have become the pivot around which the growth of any company revolves, it has become extremely important for companies to deploy appropriate customer relationship management (CRM) software solutions. However, customer requirements are diverse, and most companies do not find a suitable CRM solution that best fits their needs. In this situation, Salesforce has emerged as a market leader. With flexibility and reliability as its primary features, Salesforce is most likely to continue dominating the marketplace in the coming years.

If you are a developer, you know the pain of following numerous code components in order to create an application. Well, Salesforce comes to your rescue. With Salesforce, you can create applications in a few days or weeks and make them visible to as many clients as you want with the provided customizations.

Why Should We Learn Salesforce?

Salesforce started as a Software-as-a-Service (SaaS) company providing a CRM platform on the cloud. Along with SaaS, Salesforce provides a platform where its clients can create their own applications. This is called Platform-as-a-Service (PaaS). Although most of Salesforce’s revenue comes through PaaS, SaaS is also used by many Fortune 500 companies.

Salesforce CRM

Salesforce CRM helps companies manage relationships with their customers and integrate with other important systems. Salesforce CRM is a SaaS tool that aids companies in creating personalized solutions for their marketing, services, sales, and e-commerce problems. All Salesforce CRM data is cloud-based, so it can be accessed remotely from anywhere.

What are Salesforce Lightning Components?

The Salesforce Lightning Components are user interface (UI) frameworks that can be used by Salesforce developers for building interactive desktop applications and mobile technologies. The Salesforce Lightning components can be reused and are self-contained.

Basically, the lightning framework was created by Salesforce to develop endless applications and technologies. A few of the common features are:

  • These lightning components have ‘.cmp’ as their extensions.
  • The Salesforce Lightning component framework uses the concept of ‘web components’.
  • The languages used for building the Salesforce lightning apps are Apex, Html, CSS, and JavaScript.

Using Salesforce Lightning Components

Salesforce Lightning Components can be used to build stand-alone applications which can also be hosted on Salesforce. The lightning framework is also being used by the Salesforce CRM for handling high demand and streamlining processes between businesses, clients, and IT. This has been possible only because lightning components can be used to customize the Salesforce environment in many different ways.

How to Create Salesforce Lightning Components

  1. First, you need to log into your Salesforce developer account.
  2. After logging in, move the cursor to Developer Console and select.
  3. After navigating to the Developer Console, go to File —> New —> Lightning Component.
  4. Enter a desired name for the lightning component.
  5. Enter a description of the component and click on the Save option.
  6. Once saved, add the configurations to the saved component. Note that you have the liberty to select multiple options in the Component Configuration section.
  7. Finally, click on ‘Submit’ to create the lightning component.

Lightning Component Bundle

A Lightning Component Bundle is a set of all the necessary resources related to a component such as javascript controller, javascript helper, renderer, CSS styles, etc.

Helper

Helper consists of the javascript code for the component and its function is to segregate the calls to the server. Helper is reusable as its defined methods can be called from the controller and the renderer.

Component

Component consists of both Html codes and some lightning tags.

Controller

The Controller contains the javascript code for the component. It is required for calling the action events which are declared in the lightning component.

Renderer

The Renderer carries the code that you need to execute before or after your component elements are rendered. Types of rendering behavior include: Render, AfterRender, Rerender, and Unrender.

SVG

SVG (Scalable Vector Graphics), is a resource used in Lightning App Builder or Community Builder.

Style

The ‘Style’ resource consists of the custom style classes which you will be using in your component.

Design

For creating attributes to be used at the design time, you have to use the ‘Design’ resource. These attributes are used while working on the Lightning App builder tool.

Salesforce Architecture

The architecture of Salesforce is designed to be scalable, secure, and multi-tenant. It consists of different layers:

  • Multi-tenant Architecture: Multiple customers share the same infrastructure.
  • Metadata-driven: The behavior of the application is driven by metadata.
  • API Access: Everything in Salesforce is accessible via APIs.

Objects are essentially database tables in Salesforce. They store data.

Standard Objects: Pre-built objects like Account, Contact, Lead.

Custom Objects: Objects you create to store information specific to your organization.

Creating Custom Fields

Custom fields allow you to capture specific data points on objects.

Navigate to: Object Manager > [Select Object] > Fields & Relationships > New.

Reports and Dashboards

Reports: Lists of data based on specific criteria.

Dashboards: Visual representations of report data, showing key metrics at a glance.

Wrapper Class in Salesforce

A wrapper class is a custom class defined by a developer to display different objects on a Visualforce page or Lightning component.

It acts as a container to hold multiple data types in a single unit.

Governor Limits in Salesforce

Because Salesforce is a multi-tenant environment, it enforces limits to ensure no single organization monopolizes shared resources.

Examples include limits on SOQL query rows (50,000), CPU time, and heap size.

Process Builder

Process Builder is a point-and-click tool that lets you easily automate if/then business processes.

It is more powerful than Workflow Rules and allows for multiple actions in a single process.

Workbench in Salesforce

Workbench is a powerful, web-based suite of tools designed for administrators and developers to interact with Salesforce organizations.

It is commonly used to run SOQL queries, REST API calls, and migrate data.

Batch Apex

Batch Apex is used to process large numbers of records that would exceed normal processing limits.

It breaks the processing into manageable chunks to avoid hitting Governor Limits.

Workflow in Salesforce

Workflow rules automate standard internal procedures and processes to save time across your organization.

They consist of: 1) Criteria (If), 2) Actions (Then).

Triggers in Salesforce

Apex triggers enable you to perform custom actions before or after events to records in Salesforce, such as insertions, updates, or deletions.

Syntax: trigger TriggerName on ObjectName (trigger_events) { ... }
