Salesforce Introduction

As customers have become the pivot around which the growth of any company revolves, it has become extremely important for companies to deploy appropriate customer relationship management (CRM) software solutions. However, customer requirements are diverse, and most companies do not find a suitable CRM solution that best fits their needs. In this situation, Salesforce has emerged as a market leader. With flexibility and reliability as its primary features, Salesforce is most likely to continue dominating the marketplace in the coming years.

If you are a developer, you know the pain of following numerous code components in order to create an application. Well, Salesforce comes to your rescue. With Salesforce, you can create applications in a few days or weeks and make them visible to as many clients as you want with the provided customizations.

Why Should We Learn Salesforce?

Salesforce started as a Software-as-a-Service (SaaS) company providing a CRM platform on the cloud. Along with SaaS, Salesforce provides a platform where its clients can create their own applications. This is called Platform-as-a-Service (PaaS). Although most of Salesforce’s revenue comes through PaaS, SaaS is also used by many Fortune 500 companies.

Salesforce CRM

Salesforce CRM helps companies manage relationships with their customers and integrate with other important systems. Salesforce CRM is a SaaS tool that aids companies in creating personalized solutions for their marketing, services, sales, and e-commerce problems. All Salesforce CRM data is cloud-based, so it can be accessed remotely from anywhere.

What are Salesforce Lightning Components?

The Salesforce Lightning Components are user interface (UI) frameworks that can be used by Salesforce developers for building interactive desktop applications and mobile technologies. The Salesforce Lightning components can be reused and are self-contained.

Basically, the lightning framework was created by Salesforce to develop endless applications and technologies. A few of the common features are:

  • These lightning components have ‘.cmp’ as their extensions.
  • The Salesforce Lightning component framework uses the concept of ‘web components’.
  • The languages used for building the Salesforce lightning apps are Apex, Html, CSS, and JavaScript.

Using Salesforce Lightning Components

Salesforce Lightning Components can be used to build stand-alone applications which can also be hosted on Salesforce. The lightning framework is also being used by the Salesforce CRM for handling high demand and streamlining processes between businesses, clients, and IT. This has been possible only because lightning components can be used to customize the Salesforce environment in many different ways.

How to Create Salesforce Lightning Components

  1. First, you need to log into your Salesforce developer account.
  2. After logging in, move the cursor to Developer Console and select.
  3. After navigating to the Developer Console, go to File —> New —> Lightning Component.
  4. Enter a desired name for the lightning component.
  5. Enter a description of the component and click on the Save option.
  6. Once saved, add the configurations to the saved component. Note that you have the liberty to select multiple options in the Component Configuration section.
  7. Finally, click on ‘Submit’ to create the lightning component.

Lightning Component Bundle

A Lightning Component Bundle is a set of all the necessary resources related to a component such as javascript controller, javascript helper, renderer, CSS styles, etc.

Helper

Helper consists of the javascript code for the component and its function is to segregate the calls to the server. Helper is reusable as its defined methods can be called from the controller and the renderer.

Component

Component consists of both Html codes and some lightning tags.

Controller

The Controller contains the javascript code for the component. It is required for calling the action events which are declared in the lightning component.

Renderer

The Renderer carries the code that you need to execute before or after your component elements are rendered. Types of rendering behavior include: Render, AfterRender, Rerender, and Unrender.

SVG

SVG (Scalable Vector Graphics), is a resource used in Lightning App Builder or Community Builder.

Style

The ‘Style’ resource consists of the custom style classes which you will be using in your component.

Design

For creating attributes to be used at the design time, you have to use the ‘Design’ resource. These attributes are used while working on the Lightning App builder tool.

Data-driven Messaging in Salesforce Marketing Cloud

SFMC is able to deliver messages according to the current customer data. It can then trigger more messages according to the changes in the data and interactions.

Contact Data

Contact data covers the data related to the contact. If there is a change in contact data, SFMC will automatically decide the journey it will take. For example, if a customer was earlier listed under prospective clients and then they become a client, the data listed for them in SFMC will change accordingly.

Journey Data

Journey data covers the data related to a customer’s journey. For example, SFMC can detect if an email has been opened or interacted with. If a customer has not opened an email, it can be resent to them in a few days.

Multi-channel Messaging

SFMC includes a Journey Builder that lets users create a Journey consisting of SMSs and emails. Some use cases include:

  • Support and service: Use emails for support-related messages and SMS for timely information.
  • Post-purchase: Share all post-purchase information via email and SMS.

Salesforce Architecture – A Deep Dive

By now, you would have understood that you can use Salesforce to deliver a highly customizable application to your customers. In order to understand how Salesforce works, it is important to understand the architecture of Salesforce.

First, let’s see some important points to get introduced to Salesforce:

  • Salesforce is a cloud company, and it offers a trusted and multi-tenant cloud.
  • The Salesforce platform is the foundation of Salesforce services. It is powered by metadata and is made up of several different parts, such as data services, Artificial Intelligence, and robust APIs for development.
  • All apps sit on top of the Salesforce platform. Its pre-built offerings, namely, Salesforce Marketing Cloud and Sales Cloud, along with apps you build using the platform, have consistent, powerful functionality.
  • Everything is integrated with Salesforce. The platform technologies such as Salesforce Einstein predictive intelligence and the lightning framework are built according to the user’s convenience.

Core Architecture of Salesforce

Think about the Salesforce architecture as a series of layers sitting on top of each other. Before discussing the Salesforce architecture, let’s first go through the terminology:

Terminologies Used in Salesforce Architecture

App

All metadata elements such as objects, Visualforce pages, classes, etc. are independent of an app. An app simply helps you cluster things visually. You can have similar tabs and VF Pages in multiple apps.

Instance

An instance of Salesforce is the particular configuration that you see when you log in. A Salesforce instance basically indicates server details for a particular Salesforce organization on which it lives. It is possible for many Salesforce instances to live on one server.

Superpod

Superpod is the arrangement of frameworks and stack balancers, including outbound intermediary servers, system and capacity foundations, mail servers, SAN texture, and various other frameworks supporting different instances.

Org (Organization)

It is a single consumer of the Salesforce application. Every trial that begins on www.salesforce.com or developer.force.com produces a new org. An org is incredibly adjustable and has very clear security and sharing settings.

Sandbox

Whenever a sandbox is created, Salesforce copies the metadata from your production org to the sandbox org. By doing so, you can create multiple copies of your production org in separate environments.

There are four kinds of Salesforce sandbox:

  1. Developer Sandbox
  2. Developer Pro
  3. Partial Copy
  4. Full Sandbox

Different Layers of Salesforce Architecture

1. Multi-tenant in Salesforce Architecture

In Salesforce, all data is stored in a single database schema for all its customers.

  • It is economical because resources and maintenance are shared.
  • There can be a single instance of a software server with multiple tenants.
  • The provider has to update only one application, and the changes will get updated for all clients.

In a multi-tenant architecture, there is one common application serviced to multiple clients. The developer can create a Salesforce Application, upload it onto the cloud, and share it with multiple clients. The major advantage is that it becomes cost-effective. The development and maintenance costs can be shared at once.

2. Metadata in Salesforce Architecture

Salesforce platform uses a metadata-driven development model.

  • Developers need to concentrate only on building applications.
  • Functionalities of an app is defined as metadata in the Salesforce database.
  • This increases developers’ productivity.

The issues of security get sorted with this particular architecture since here all data is stored on different levels, i.e., data above data. All the data is stored in shared databases; the metadata points to one particular clients’ data in a shared database.

Consider three clients, A, B, and C, sharing a single database. These clients will have access to their own metadata from the shared data. This ensures that you don’t get into someone else’s metadata, increasing security.

3. CRM and Related Functionality

In its list of features, Salesforce includes all aspects of CRM functionality and also provides features for the creation of apps, integrating analytics, and more.

4. Enterprise Ecosystem

The enterprise ecosystem of Salesforce is huge. There are contributions from a large number of partners by creating and maintaining applications in this platform. This ecosystem is created on top of Salesforce’s unique business model that is focused on engagement and collaboration.

5. API

A powerful source of APIs is provided by Salesforce. This APIs help in the development and customization of the Salesforce mobile application. These APIs enable diverse bits of programming to interface with each other and trade data. These APIs are comparable. Without knowing many details, you can connect your apps with other apps.

Salesforce design isn’t just a random series of trial-and-error experiments. Rather, each and every feature of its design has been planned and placed precisely.

Different DataTypes in Salesforce

Apex in Salesforce assigns a data type to all variables and expressions, such as sObject, primitive, or enum. We use these data types as required depending on the condition.

  • Primitives: Integer, Double, Long, Date, Datetime, String, ID, or Boolean.
  • sObject: General or specific objects such as an Account, Contact, or MyCustomObject__c.
  • Collections:
    • A list (or array) of primitives, sObjects, user-defined objects, Apex classes created objects, or collections.
    • A set of primitives.
    • A map from a primitive to a primitive, sObject, or collection.
  • Enum: A typed list of values.
  • User-defined Objects: Objects made from Apex classes defined by the user.
  • System Objects: Objects built with Apex classes provided by the system.
  • Null: For the null constant.

Data Types in API

There are different data types depending on the circumstances. Here are the five in API:

  1. ID
  2. String
  3. Boolean
  4. Double
  5. Varies by type

Variables in Salesforce

  • Declaring and initializing local variables with one line, similar to Java, is possible here.

How to Create Reports and Dashboards in Salesforce

What is a Salesforce Report?

A report contains data in an organized form. Salesforce reports contain data that is organized according to predefined criteria. The data that is uploaded on Salesforce is used to generate reports. Reports on Salesforce can be shared according to the user’s needs. There are various types of folders available to store Salesforce Reports which decide the accessibility of that report. What is a Dashboard?

A dashboard is used to see a pictorial representation of reports. Organizing visual data together helps in understanding them faster. You can use the dashboard to show the reports in various forms of charts like pie charts, funnel charts, etc. You can add different components to a Dashboard. Each of these components will correspond to a particular report. A report can be a part of two or more components.

How to Create Reports in Salesforce?

It is very simple and quick to create reports in Salesforce. First, log in to your Salesforce association group and then click on Reports.

  • Create New Report folder: Use this to create a new report folder.

Types of Salesforce Reports

The following are the four types of reports in Salesforce:

1. Tabular Report

Tabular reports are the basic and most simple form of Salesforce reports. They are especially useful in cases when you need to export data. They show data effectively in rows and a tabular form. It is suggested not to use Tabular reports when manipulating data (calculations, totals).

2. Summary Report

Summary reports are basically used to show groups of data. They are the most commonly used reports. You can group data in the name of different accounts and perform calculations (total, averages, max, min). Grouping is mostly done in rows.

3. Matrix Report

Matrix reports show data in rows as well as columns. They are complex and used specifically to display large amounts of complex data. Unlike Summary reports (rows only), Matrix reports group data in both rows and columns.

4. Joined Report

By using Joined reports, you can create two separate reports and compare their individual data.

Wrapper Class in Salesforce

A wrapper class is nothing but a collection of different Salesforce data types. In Salesforce, you can combine multiple data types and utilize them for various purposes. For example, there is a wrapper class that can access the account records and display them in an in-page block table. You have an option of a checkbox, and you can associate multiple serial numbers to display the number of records.

When elucidating the architecture of Salesforce, you can see that it is somewhat similar to a database wherein the records are defined as rows, the fields are determined as columns, and the objects are defined as tables. Having different collections of classes assigned to facilitate a successful development process, Salesforce possesses the MVC architecture.

Wrapper Class in Salesforce as a Solution

A wrapper class is used mainly to wrap data collected from the present objects to a new object. A sObject is defined as a distinct type that doesn’t include a database or any relevant field; it doesn’t have any API page but is definitely a valid object.

In other orders, a wrapper class is a container class; it is a data structure and an abstract data type that contains various objects and their members together. A wrapper class is also called a custom object which is determined by a developer with a particular set of properties. If you want to wrap various data objects, there is no better option than wrapper classes in Salesforce.

Wrapper Class Use Case in LWC

Following is a wrapper class use case in LWC:

  1. Display the list of Open Cases in the form of DataTable.
  2. Ensure that the user is given the ability to select multiple cases at a time.
  3. Create a “Close Selected Cases” button.
  4. The ‘Close Selected Cases’ button should close all the selected cases once clicked.

Advantages of a Wrapper Class in Salesforce

  • When considering the JSON structure, the wrapper class structure is comparable to the data visualization technique on a particular web page.
  • There isn’t any requirement for passing the map structure for browsing elements or for managing the relationship between objects.
  • If you want to pass a sObject, there isn’t any penalty for the same, and it can also be extendable to the class constructors.
  • Data can be organized perfectly when it is nested.
  • A wrapper class and a container class can be kept together, but it is better if they are stored separately. When kept independently, it is easy to maintain and reuse them, preventing code duplication.

Therefore, a wrapper class is used to construct a new object within the Apex code which can consolidate a set of fields required at runtime or various fields from different objects. This provides extra control and flexibility.

Wrapper Class as an Everyday Tool in Salesforce

All Salesforce Developers use wrapper classes on a daily basis. This is normal in Visualforce wherein queries are performed on the data before presenting it to the user.

A few activities included are:

  • Connecting values concurrently for various fields of an object using a mapping table.
  • Presenting various threads of data mutually in a single list.
  • Enhancing a data object by extracting the data from another object.

A wrapper class doesn’t have an inner class determined by default, yet the practice of having an inner class can be more efficient and simple in Salesforce. A wrapper class is used similarly as unions, structures, or algebraic types for any other class.

MVC Architecture

The MVC (Model View Controller) architecture in Salesforce is a software pattern that separates the representation of information from the user’s interaction with it. Salesforce UI has an MVC model, and here Salesforce metadata is stored to view it as XML later.

The MVC architecture contains the following three modules:

  • Model (Database Layer): The schema and data that Salesforce uses to represent the system completely. sObjects can be considered the model as every entity is mapped to some sObject.
  • View (UI Layer): It is how the schema and data are represented. Visualforce is used to present the data to the users.
  • Controller (Business Logic Layer): A controller is basically an Apex Class extended to the controller; data is relinquished to accessible models first to view the elements.

Handling Nested Structures

Developers are limited to a data structure of a particular object type while the relationship between these data structures is necessary. Controllers are responsible for modeling and viewing objects, components, and pages.

As seen in standard architecture, a single contact can have many orders wherein each order can have various order lines. This kind of nested structure is exemplified through a JSON or NoSQL database.

The Verdict

A wrapper class is a proven enhancement of growth for Salesforce Developers with its wider range of solutions. Whenever you want to compare custom objects or need to add more functionalities, you can try using wrapper classes. You can also add many fields within an object whenever required and maximize the usability of an application.

Governor Limits in Salesforce

Governor limits in Salesforce are used to ensure the effective use of resources on the Force.com multi-tenant platform. In order to execute the code efficiently, Salesforce.com sets some limits.

What are Governor Limits in Salesforce?

To ensure that no one acquires resources from others, Force.com sets many restrictions that limit normal code execution. Salesforce has to do this because of its multi-tenant architecture, where all organizations and customers share a single resource. If one of the governor’s limits is not met, an error will be raised and the program will stop. Therefore, it is important to ensure that our code is scalable and does not violate governor limits.

Types of Salesforce Governor Limits

  • Per-transaction Certified Managed Package Limits
  • Static Apex Limits
  • Per-transaction Apex Limits
  • Lightning Platform Apex Limits
  • Size-specific Apex Limits

Difference Between Synchronous and Asynchronous Execution

  • Synchronous: All code gets executed at once.
  • Asynchronous: A part of the process gets executed in the background as a separate thread. Asynchronous execution has bigger governor limits.

How to Avoid Salesforce Governor Limits

As a developer, ensure code is scalable. Follow this process:

  • Do not have DML statements or SOQL queries in our FOR loop.
  • Try not to use SOQL or DML operations in the loop.
  • Try to bulkify the code and helper methods.
  • Query large data sets.
  • Use Batch Apex if we want to process 50,000 records.
  • Streamline various triggers on the same object.

Conclusion

We hope you now understand how the governor limits work in Apex and how to avoid them. When working on code, it is crucial to be aware of the limits Salesforce has set.

Process Builder in Salesforce

Salesforce Process Builder is a powerful automation tool that enables you to define the sequence of actions or evaluate criteria for a record. It is a must-have tool for Salesforce developers.

What is Process Builder Salesforce?

Process Builder offers a range of eight actions that can be associated with it:

  1. Creating Records: Effortlessly generate a new record and assign specific field values.
  2. Updating Records: Update one or multiple records associated with the original record triggering the process.
  3. Posting on Chatter: Share information with other users via Chatter.
  4. Quick Action: Use global actions or objects within Salesforce on any record.
  5. Launching or Triggering the Flow: Launch a flow from your process to automate different processes.
  6. Submitting for Approval: Automatically submit the record that started your process for approval.
  7. Calling Apex Code: Invoke Apex code that you have already written.
  8. Invoking Another Process: Invoke another process from the current one.

Components of a Process

There are three major components: Trigger, Criteria, and Actions.

1. Trigger

The trigger identifies when the process should start running. It determines which objects and changes the process pays attention to:

  • Only when a record is created.
  • When a record is created or edited.

2. Criteria

Criteria determine whether or not actions should be executed. If the record does not meet the criteria, the process skips associated actions. Criteria allow you to:

  • Set filter conditions.
  • Give a customized formula (must evaluate to true or false).
  • Opt out of criteria to always execute actions.

3. Actions

Actions determine what the process should do. When a criteria node evaluates to true, the process executes the associated actions or waits to execute them at a scheduled time.

  • Immediate Actions: As soon as criteria are evaluated as true, the action is executed immediately.
  • Scheduled Actions: Executed at a specified time (e.g., 8 days before close date). Salesforce re-evaluates criteria before executing scheduled actions.

Process Types

Process Builder supports different process types for automation:

  • Record Change: Starts the process when a record is created or edited.
  • Invocable: The process is started when called by another process.
  • Platform Event: The process starts when a platform event message is received.

Why use Salesforce Process Builder?

Process Builder is used for automating business processes without code. Key benefits include:

  • Visual breakdown lets you know what to expect at a glance.
  • Applications are developed three times faster and at half the cost.
  • Can combine multiple workflows into a single process with a chain of if-then statements.
  • Can update child records.
  • Can notify both Chatter groups and email alerts.
  • Multiple actions can be associated with each criterion.

Steps to Create a Process Builder

Scenario: Create an order record when the opportunity is Closed Won and the amount is US$5 million.

  1. Setup: Go to Setup and search for Process Builder. Click on New.
  2. Define Process: Enter Process Name (e.g., "Demo") and Description. Select "A record changes" for how to start the process. Click Save.
  3. Add Object: Click Add Object. Select "Opportunity" and choose "when a record is created or edited". Click Save.
  4. Add Criteria: Click Add Criteria. Enter Criteria Name (e.g., "Stage=Closed Won? And Amount>5M"). Choose "Conditions are met".
  5. Set Conditions:
    • Row 1: Field (Opportunity > Stage), Operator (Equals), Type (Picklist), Value (Closed Won).
    • Row 2: Field (Opportunity > Amount), Operator (Greater Than), Value (5000000).
  6. Save Criteria: Click Save after setting conditions.
  7. Add Action: Click Add Action in the Immediate Action box. Choose Action Type (e.g., Create a Record), Name the action, and set field values for the new Order record.
  8. Activate: Click Save, then click Activate to enable the process.

Conclusion

Salesforce Process Builder is a powerful tool that can be used to automate business processes. It is an important part of the entire Salesforce automation bundle.

What is Workbench in Salesforce?

Workbench Salesforce is a web-based tool designed for Salesforce Administrators and Developers. It allows them to interact with Salesforce organization or Force APIs to update, delete, insert, and export data. It is a highly adaptable and cloud-hosted toolkit.

Workbench Features

Data Management Capabilities

  • Edit: Click and edit single records.
  • ZIP File: Upload a ZIP file containing binary files and a CSV file to insert, upsert, delete, or update binary files.
  • Smart Lookup: Find relevant Salesforce IDs through the Smart Lookup feature.
  • Delete and Purge: Delete records (retrievable from recycle bin) or Purge them to delete permanently.

Data Security

  • CSRF Protection: Protects data from hackers. It is dependent on the code in Workbench.
  • SSL: Salesforce uses SSL to connect to Workbench by default.

How to Login to Workbench Salesforce?

  1. Access: Go to https://workbench.developerforce.com/login.php.
  2. Login: Log in with your Salesforce credentials. Select the desired environment and API versions. Tick the terms and conditions checkbox and hit Login with Salesforce.
  3. Jump To Menu: After logging in, you will see a Jump to menu. You can switch between different options:
    • Apex Class
    • Apex Page
    • Workflow
    • Approval
    • SOSL Query
    • SOQL Query
    • Insert, Update, Delete, Undo, Upsert

Navigating Workbench Menus

On the top menu bar, you will see five drop-down items: Info, Queries, Data, Migration, and Utilities.

1. Info

Contains three sections:

  • Standard & Custom Objects: Choose an object to view related information.
  • Metadata Types & Components: Choose components like Apex Class, Approval, Workflow, or Apex Page. All related information will be displayed.
  • Session Information: Displays current user session details (connection mode, profile ID, user info). System Administrators can end other sessions.

2. Queries

Allows you to run queries:

  • SOQL Query: Select objects and fields, apply conditions, and view records as a list or matrix. You can download results in CSV format.
  • SOSL Query: Perform text-based searches similarly to SOQL.

3. Data

Perform data operations on objects:

  • Insert, Update, Delete, Undo, Upsert.
  • Upload records from ZIP or CSV files.
  • Click View in Salesforce to be redirected to your org.

4. Migration

Retrieve packages using package.xml and deploy them directly to the Salesforce organization.

5. Utilities

  • REST Explorer: Choose HTTP methods (GET, POST, etc.) to perform REST API services.
  • Apex Execute: Create, delete, and update records or new objects using Apex scripts. New objects containing null values must be set after creation.
  • Password Management: Set and reset passwords for the Salesforce org.

Advantages of Workbench Salesforce

  • Data Management: Edit records by clicking unique IDs. Upload ZIP files for binary data operations.
  • Smart Lookup: Provides foreign or standard ID lookup fields to find relevant Salesforce IDs during insert, update, or delete.
  • Easy Data Extraction: Simple removal of information related to users, roles, profiles, and permission sets.

Conclusion

Workbench in Salesforce is a powerful and secure data management tool that is of great help to developers. It makes both data management and extraction much easier.
